# MemoDoc - 테스트 및 검증 계획 (TEST_PLAN)

---

## 가설

### 핵심 가설
> 업무 메모를 MemoDoc에 입력하면, 사람이 직접 정리하는 것보다 빠르고 실용적인 수준의 문서를 생성할 수 있다.

### 세부 가설 (검증 가능한 단위로 분해)

| ID | 가설 | 측정 방법 |
|---|---|---|
| H1 | 메모 입력 후 30초 이내에 첫 번째 탭 문서가 완성된다 | 생성 시작~완료 시간 측정 |
| H2 | 생성된 문서를 Notion에 붙여 넣었을 때 별도 수정 없이 사용 가능하다 | 사용자 수정 횟수 측정 |
| H3 | 메모 유형(회의/교육/아이디어)에 따라 요약 형식이 적절하게 달라진다 | 유형별 출력 섹션 일치율 확인 |
| H4 | Mermaid 다이어그램이 프로세스가 있는 메모에서 오류 없이 렌더링된다 | 렌더링 성공/실패 횟수 집계 |
| H5 | 사용자가 직접 문서를 작성하는 것보다 시간이 단축된다 | MemoDoc 소요 시간 vs 직접 작성 시간 비교 |

## 검증 방법

### 1. 기능 검증 (개발자 테스트)
- 정의된 테스트 시나리오를 직접 실행하여 기능 동작 확인
- 다양한 메모 유형 및 길이로 반복 테스트

### 2. 사용자 검증 (유저 테스트)
- **대상:** 소프트웨어 개발자 또는 기획자 3~5명
- **방법:** 실제 업무 메모를 제공하고 MemoDoc 사용 후 피드백 수집
- **측정 항목:**
  - 생성 문서 품질 만족도 (5점 척도)
  - 문서 수정 없이 바로 사용 가능 여부 (Y/N)
  - 직접 작성 대비 시간 절감 체감 (5점 척도)
  - 가장 유용한 탭 / 가장 불필요한 탭 선택

### 3. 정량 측정
- 탭별 생성 소요 시간 (ms 단위 측정)
- Mermaid 렌더링 성공률 (성공 횟수 / 전체 시도)
- 오류 발생 시 재시도 성공률

## 테스트 시나리오 및 실행 결과

| 항목 | 입력 | 기대 결과 | 결과 |
|---|---|---|---|
| API 키 저장 | 유효한 `sk-or-` 키 | localStorage 저장, 새로고침 후 유지 | ✅ |
| 잘못된 API 키 형식 | `sk-or-` 아닌 값 | 오류 메시지, 앱 정상 동작 | ✅ |
| 구 Gemini 키 자동 삭제 | `AIza...` 키 localStorage 존재 | 자동 삭제 후 재입력 유도 | ✅ |
| 빈 입력 생성 | 메모 없이 버튼 클릭 | 경고 메시지 표시 | ✅ |
| 짧은 메모 차단 | 10자 미만 입력 | "너무 짧아요" 경고 | ✅ |
| 6탭 순차 생성 | 회의 메모 텍스트 | 탭별 문서 스트리밍, 진행률 표시 | ✅ |
| 메모 유형 분기 - 회의 | 회의 선택 + 회의 메모 | 결정 사항 / 다음 할 일 섹션 | ✅ |
| 메모 유형 분기 - 교육 | 교육 선택 + 학습 메모 | 핵심 개념 / 배운 점 / 적용 포인트 | ✅ |
| 도식화 - 프로세스 있음 | 단계 흐름 포함 메모 | Mermaid 플로우차트 렌더링 | ✅ |
| 도식화 - 프로세스 없음 | 개념 설명 메모 | "도식화할 프로세스가 없습니다" 출력 | ✅ |
| 오류 시 재시도 | rate limit 오류 발생 탭 | 🔄 재시도 버튼으로 단일 탭 재생성 | ✅ |
| 내보내기 | 생성된 탭 클릭 | .md 파일 다운로드 | ✅ |

## 성공 기준 및 측정 방법

| 기준 | 목표값 | 측정 방법 | 실측값 | 달성 |
|---|---|---|---|---|
| 첫 탭 생성 완료 시간 | 30초 이내 | 브라우저 콘솔 타이머 | 약 20~30초 | ✅ |
| 전체 6탭 생성 완료 시간 | 3분 이내 | 생성 시작~완료 시간 측정 | 약 2~3분 | ✅ |
| Mermaid 렌더링 성공률 | 80% 이상 | 성공/전체 횟수 집계 | 약 85% | ✅ |
| 내보낸 .md Notion 호환 | 수정 없이 사용 가능 | 실제 Notion 붙여넣기 확인 | 마크다운 구조 정상 렌더링 | ✅ |
| 사용자 만족도 | 4점/5점 이상 | 5점 척도 설문 | 미수집 (예정) | ⬜ |
| 직접 작성 대비 시간 절감 | 50% 이상 | 직접 작성 시간과 비교 | 미측정 (예정) | ⬜ |

## 사용자 피드백 수집 계획

### 설문 항목 (5점 척도)
1. 생성된 문서의 품질이 실무에서 바로 사용할 수 있는 수준인가?
2. 직접 정리하는 것보다 시간이 절약되었는가?
3. 6개 탭 중 실제로 유용하게 사용할 탭은 몇 개인가?
4. 다시 사용할 의향이 있는가?

### 정성 피드백 항목
- 가장 유용했던 탭과 그 이유
- 가장 아쉬웠던 점
- 추가되었으면 하는 기능

## 발견된 이슈 및 조치

| 이슈 | 원인 | 조치 |
|---|---|---|
| 6탭 병렬 요청 시 rate limit | 무료 모델 분당 한도 초과 | 순차 처리 + 3초 딜레이 |
| Mermaid 파싱 오류 | 이모지·줄바꿈·괄호 포함 노드 | 프롬프트에 노드 작성 규칙 명시 |
| 교육 메모에 회의 형식 출력 | 소형 모델의 조건부 지시 미준수 | 유형 선택 UI로 프롬프트 강제 분기 |
| 전체 생성 시간 2~3분 | 순차 처리 + 무료 모델 응답 지연 | Phase 2에서 유료 모델 전환 시 개선 예정 |
