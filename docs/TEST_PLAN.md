# MemoDoc - 테스트 계획 (TEST_PLAN)

---

## 가설

업무 메모를 입력하면 AI가 즉시 6가지 형태의 유용한 문서를 생성할 수 있다.
사람이 직접 정리하는 것보다 빠르고, 결과물의 품질이 충분히 실용적이다.

---

## 검증 방법

- 실제 회의 메모 샘플로 6탭 전체 생성 후 품질 직접 확인
- Notion에 붙여 넣었을 때 별도 수정 없이 사용 가능한지 체크
- Mermaid 다이어그램이 문법 오류 없이 렌더링되는지 확인

---

## 테스트 시나리오

| 항목 | 입력 | 기대 결과 |
|---|---|---|
| API 키 저장 | 유효한 `sk-or-` 키 | localStorage 저장, 새로고침 후 유지 |
| 잘못된 API 키 | `sk-or-` 아닌 값 | 오류 메시지, 앱 정상 동작 |
| 빈 입력 생성 | 메모 없이 버튼 클릭 | 경고 메시지 표시 |
| 6탭 생성 | 회의 메모 텍스트 | 탭별 문서 스트리밍 출력 |
| 도식화 탭 | 프로세스 포함 메모 | Mermaid 플로우차트 렌더링 |
| 내보내기 | 생성된 탭 클릭 | .md 파일 다운로드 |
| 짧은 메모 | 한 문장 | 에러 없이 결과 생성 |
| 긴 메모 | 10,000자 | 정상 처리 또는 명확한 오류 |

---

## 성공 기준

- 첫 텍스트 출력까지 3초 이내
- 6탭 모두 오류 없이 완료
- 내보낸 .md가 Notion에서 그대로 사용 가능
- Mermaid 다이어그램 렌더링 성공률 80% 이상

---

## 기대 효과

- 회의 직후 문서화 시간 10분 → 1분 이하
- 다이어그램 작성 부담 제거
- 메모가 팀 공유 자산으로 즉시 전환 가능

---

## 테스트 샘플 메모

```
오늘 팀 회의에서 MSA 전환 논의.
3단계 계획: 도메인 분리 → API Gateway → 이벤트 버스
결정: 6개월 로드맵. 첫 번째로 User 서비스 분리.
담당: 백엔드팀. 다음 회의: 2주 후.
```
