# MemoDoc - PRD

> 작성일: 2026-03-13 | 버전: 1.0

---

## 배경

업무 중 작성한 메모는 대부분 개인 노트에서 멈춘다. 나중에 꺼내보면 맥락 없이 단어만 나열되어 있고, 팀에 공유하거나 설계에 재활용하기 어렵다.

---

## 문제 정의

- 회의 메모를 공식 문서로 정리하는 데 시간이 너무 걸린다
- 다이어그램이나 플로우차트 작성 진입 장벽이 높다
- 메모가 지식 자산으로 이어지지 않는다

### 문제 규모

- 국내 사무직 근로자 약 **1,200만 명** 중 정기 회의 참여자 추정 **600만 명+**
- 주 평균 회의 횟수: **4.6회** (한국생산성본부, 2023)
- 회의록 작성에 소요하는 시간: 평균 **30~60분/회**
- 회의록을 작성하지 않거나 개인 메모로만 보관하는 비율: **약 70%** (미작성 손실)
- **타겟 시나리오:** 주 5회 회의하는 개발자 기준, MemoDoc 도입 시 주당 **2~4시간** 절감 가능

---

## 목표

업무 메모를 붙여 넣으면 AI가 즉시 6가지 형태의 지식 문서를 자동 생성해 준다.

---

## 사용자 페르소나

### 페르소나 A — 백엔드 개발자 (주요 타겟)
- **역할:** 5년차 백엔드 개발자, 주 3회 스프린트/기술 회의 참여
- **통증점:** 회의 후 노션 회의록 작성에 30~60분 소요. 퇴근 전 밀린 메모 정리가 반복적 스트레스
- **목표:** 회의 직후 5분 안에 공유 가능한 회의록 완성
- **사용 시점:** 회의 종료 직후, 슬랙에 공유 전

### 페르소나 B — 프로덕트 매니저
- **역할:** 스타트업 PM, 이해관계자 보고 및 요구사항 문서 작성 담당
- **통증점:** 회의 메모를 기획서 형식으로 변환하는 과정이 번거롭고, 다이어그램 도구(draw.io 등) 학습 비용이 높음
- **목표:** 카카오톡·메모앱 수준의 메모에서 즉시 보고용 문서 생성
- **사용 시점:** 요구사항 정의 회의 직후, 설계 문서 작성 전

### 페르소나 C — 기술 학습자 (부가 타겟)
- **역할:** 온라인 강의 수강 중인 주니어 개발자
- **통증점:** 강의 필기가 흩어진 메모 수준에서 머물고, 나중에 복습하기 어려움
- **목표:** 학습 메모를 정리된 개념 문서로 자동 변환
- **사용 시점:** 강의 수강 직후

## 기능 요구사항

| ID | 기능 | 설명 |
|---|---|---|
| FR-1 | 메모 입력 | 자유 형식 텍스트 입력 (최대 10,000자) |
| FR-2 | 요약 | 핵심 내용 bullet point 정리 |
| FR-3 | 개념 정리 | 주요 용어 추출 + 정의 |
| FR-4 | 구조화 문서 | 마크다운 공식 문서 생성 |
| FR-5 | 도식화 | Mermaid 플로우차트 자동 생성 |
| FR-6 | 단계별 설명 | Step-by-step 가이드 |
| FR-7 | 설계 활용 포인트 | 재활용 가이드 및 체크리스트 |
| FR-8 | 내보내기 | .md 파일 다운로드 (탭별 / 전체) |

## 범위

**포함:** 브라우저 단독 실행, AI 문서 생성 6종, 마크다운 렌더링, 다이어그램 렌더링, 내보내기, 다크모드

**미포함:** 서버, 로그인, 문서 저장/히스토리, 팀 공유

---

## 사용자 시나리오

1. 브라우저에서 `hackathon.html` 열기
2. OpenRouter API 키 입력 후 저장
3. 회의 메모를 텍스트 영역에 붙여 넣기
4. "문서 생성" 클릭 → 6개 탭이 동시에 스트리밍으로 채워짐
5. 원하는 탭의 결과를 .md로 내보내기

## MVP 범위

**포함 (Must Have)**
- `hackathon.html` 단일 파일로 서버 없이 브라우저 실행
- OpenRouter API 키 입력 → localStorage 저장 → 재사용
- 메모 유형 선택 (회의/교육/아이디어) → 유형별 적응형 요약
- 6탭 순차 생성 + 스트리밍 출력 + 탭별 재시도
- Mermaid 다이어그램 렌더링 (프로세스 없으면 생략)
- .md 내보내기 (탭별 / 전체)
- 다크모드, 반응형 레이아웃, 클립보드 복사

**제외 (Out of Scope)**
- 서버/백엔드, 로그인/회원가입
- 문서 저장 및 히스토리 관리
- 팀 공유, 협업 기능
- 모델 선택 UI (상수로 관리)
